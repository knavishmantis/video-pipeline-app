4. Next Steps

1
Set up PostgreSQL training database
Schema for videos, scenes, labels. Can live in idea-research repo or as a shared service.
2
Bulk transcript + video scraper
Extend idea-research collectors: yt-dlp for video downloads, YouTube API for transcripts + view counts. Store in DB. Target 5-10K from top channels.
3
Gemini video processing pipeline
Feed each downloaded video to Gemini 2.5 Flash → extract scene-by-scene visual descriptions → store in DB scenes table. ~$40 for 5K videos.
4
Auto-label with Claude
For each transcript: classify topic, hook type, structure pattern, quality tier. Store labels in DB. ~$15-30.
5
Training data formatter + first fine-tune
Query DB for top 20% by views. Format as JSONL: {topic + angle + scene structure} → {transcript}. Fine-tune GPT-4o-mini (~$25). Evaluate.
6
Build agent chain
Researcher → Angle → Writer → Scene Mapper → Critic. Each agent queries DB for context. Writer uses fine-tuned model. CLI: python -m script_engine.run --topic "Looting"
7
Connect to video-pipeline-app
Approved scripts push via MCP/API → create short with scenes, preset clips, directions. Close the loop. Published performance feeds back to DB.
